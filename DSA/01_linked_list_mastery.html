// 00 — PREREQUISITE TOPICS

Before You Begin

These are the foundational concepts you need to understand before diving into linked lists. Each one is explained below.

📦 Object / Class

A class is a blueprint. An object is a real thing made from that blueprint. Example: Node myNode = new Node(5); — here Node is the class, myNode is the object.

🔗 Reference (Pointer)

In Java, variables for objects hold a reference — like a street address to where the object lives in memory. Not the object itself, but directions to find it.

🧠 Memory (Heap vs Stack)

Stack: fast, small, stores local variables. Heap: large, stores objects. When you do new Node(), the node goes on the heap.

🎭 null

null means "points to nothing." In linked lists, the last node's next pointer is set to null to signal the end of the chain.

🔁 Iteration / Traversal

Moving through a sequence one step at a time. In arrays: arr[i]. In linked lists: following node.next until you reach null.

📐 Generic Types <T>

Java generics let you write code that works with any data type. LinkedList<Integer> stores integers; LinkedList<String> stores strings — same class, different type.

🧩 Big O Notation

A way to measure how an algorithm scales with input size. O(1) = instant, O(n) = grows linearly with size, O(n²) = slow for large inputs.

🏗️ Data Structure

A way of organizing data so it can be stored and accessed efficiently. Arrays, lists, stacks, queues, trees — each solves different problems differently.

💡

Data Engineer tip: Think of a reference/pointer like a foreign key in a database — it doesn't store the actual data, it stores the address to find that data. Linked lists are essentially a chain of "records with foreign keys pointing to the next record."

// 01 — FOUNDATION

What is a Linked List?

A linked list is a sequence of nodes where each node stores data and a pointer to the next node. Unlike arrays, nodes are scattered in memory — not stored in one contiguous block.

🏠 Analogy: Treasure Hunt

Imagine a treasure hunt where each clue tells you the data (treasure at this spot) and the address of the next clue. You can only find clue #3 by reading clue #2 first. You can't skip directly to clue #7. That's a linked list.

▸ Anatomy of a Single Node
Node.java — The Building Block
// A Node is the single building block of a Linked List.
// Think of it as one "record" in a chain.

public class Node<T> {

    public T data;          // stores the actual value (any type)
    public Node<T> next;   // reference (address) to the next node

    // Constructor: creates a node with given data, next = null by default
    public Node(T data) {
        this.data = data;
        this.next = null;   // starts unlinked
    }
}

// Usage:
Node<Integer> first  = new Node<>(10);
Node<Integer> second = new Node<>(20);
Node<Integer> third  = new Node<>(30);

first.next  = second;  // 10 → 20
second.next = third;   // 20 → 30
// third.next is already null → end of list
// 02 — TYPES OF LINKED LISTS

The Four Types

Each type is suited to different use cases. Understanding which to use is a key engineering decision.

1️⃣
Singly Linked List
Each node points only FORWARD. The most basic type.
AccessO(n) Insert (head)O(1) Insert (tail)O(n) SpaceO(n)
2️⃣
Doubly Linked List
Each node has both NEXT and PREV pointers. Traverse in both directions.
Insert (any)O(1)* Delete (any)O(1)* Extra space2× pointers

* O(1) only if you already have a reference to the node; otherwise O(n) to find it.

3️⃣
Circular Linked List
The last node's NEXT points back to the HEAD. No null end. Useful for round-robin scheduling.

Quick Comparison

TypeTraversalMemoryBest Use Case
SinglyForward onlyLowStacks, simple queues
DoublyBoth waysMediumUndo/redo, browser history
CircularLoop foreverLowRound-robin, OS scheduling
Doubly CircularBoth + loopHighAdvanced data structures
// 03 — CORE OPERATIONS

Core Operations in Java

The full implementation of a Singly Linked List — understanding every operation step by step.

SinglyLinkedList.java — Complete Implementation
public class SinglyLinkedList<T> {

    private Node<T> head;   // always points to the first node
    private int size;       // tracks count for O(1) size check

    // ── 1. INSERT AT HEAD ─────────────────────────────────
    // O(1) — Just re-wire the head pointer
    public void addFirst(T data) {
        Node<T> newNode = new Node<>(data);  // 1. create new node
        newNode.next = head;                    // 2. new → old head
        head = newNode;                         // 3. head = new node
        size++;
    }

    // ── 2. INSERT AT TAIL ─────────────────────────────────
    // O(n) — Must walk to the end first
    public void addLast(T data) {
        Node<T> newNode = new Node<>(data);
        if (head == null) { head = newNode; size++; return; }
        Node<T> current = head;
        while (current.next != null) {  // walk until last node
            current = current.next;
        }
        current.next = newNode;             // attach new node at end
        size++;
    }

    // ── 3. DELETE BY VALUE ────────────────────────────────
    // O(n) — Find the node before the target, then bypass it
    public void delete(T target) {
        if (head == null) return;
        if (head.data.equals(target)) { // target is the head itself
            head = head.next;
            size--; return;
        }
        Node<T> prev = head;
        while (prev.next != null) {
            if (prev.next.data.equals(target)) {
                prev.next = prev.next.next;  // skip over the target node
                size--; return;
            }
            prev = prev.next;
        }
    }

    // ── 4. SEARCH / CONTAINS ──────────────────────────────
    // O(n) — Linear scan from head
    public boolean contains(T target) {
        Node<T> current = head;
        while (current != null) {
            if (current.data.equals(target)) return true;
            current = current.next;
        }
        return false;
    }

    // ── 5. PRINT / TRAVERSE ───────────────────────────────
    public void print() {
        Node<T> current = head;
        StringBuilder sb = new StringBuilder();
        while (current != null) {
            sb.append(current.data);
            if (current.next != null) sb.append(" → ");
            current = current.next;
        }
        sb.append(" → null");
        System.out.println(sb);
    }
}

// 04 — CODING PATTERNS

The 6 Essential Patterns

These patterns are reusable templates for solving linked list problems. Master these and 90% of interview and real-world problems become recognizable.

👆
Pattern 1: Two Pointers (Slow & Fast)
Floyd's Algorithm

Use two pointers moving at different speeds. The slow pointer moves 1 step; the fast pointer moves 2 steps. Used to detect cycles, find the middle node, or find k-th from end.

▸ Finding Middle Node with Slow/Fast Pointers
FindMiddle.java
// Pattern: Slow-Fast Pointer — Find Middle Node
public Node<Integer> findMiddle(Node<Integer> head) {
    Node<Integer> slow = head;  // moves 1 step
    Node<Integer> fast = head;  // moves 2 steps

    while (fast != null && fast.next != null) {
        slow = slow.next;        // 1 hop
        fast = fast.next.next;   // 2 hops
    }
    return slow;  // slow is now at the middle
}

// Cycle Detection (Floyd's) — same pattern
public boolean hasCycle(Node<Integer> head) {
    Node<Integer> slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true;  // they met → cycle exists
    }
    return false;  // fast hit null → no cycle
}
🔄
Pattern 2: In-Place Reversal
Three Pointer Technique

Reverse the direction of all pointers using three variables: prev, current, and next. O(n) time, O(1) space — no extra list created.

▸ Reversing Pointers In-Place
ReverseList.java
public Node<Integer> reverse(Node<Integer> head) {
    Node<Integer> prev    = null;
    Node<Integer> current = head;
    Node<Integer> next    = null;

    while (current != null) {
        next          = current.next;  // ① save next before we overwrite it
        current.next  = prev;          // ② flip the pointer BACKWARD
        prev          = current;       // ③ advance prev
        current       = next;          // ④ advance current
    }
    return prev;  // prev is now the new HEAD
}
🔀
Pattern 3: Merge Two Sorted Lists
Dummy Head Trick

Compare heads of two lists, pick the smaller one, advance that pointer. The "dummy head" simplifies edge cases — you never need to handle empty lists specially.

⚙

Data Engineer context: This is essentially a merge step in merge sort, which is the backbone of external sort in databases and big data tools like Spark. When Spark merges sorted partitions, it uses this exact algorithm!

MergeSortedLists.java
// Merge: [1→3→5] + [2→4→6] → [1→2→3→4→5→6]
public Node<Integer> mergeSorted(Node<Integer> l1, Node<Integer> l2) {
    Node<Integer> dummy = new Node<>(0);  // dummy head — skip edge cases
    Node<Integer> tail  = dummy;

    while (l1 != null && l2 != null) {
        if (l1.data <= l2.data) {
            tail.next = l1;   // pick from list 1
            l1 = l1.next;
        } else {
            tail.next = l2;   // pick from list 2
            l2 = l2.next;
        }
        tail = tail.next;    // advance tail
    }
    tail.next = (l1 != null) ? l1 : l2;  // attach remaining
    return dummy.next;  // skip dummy, return real head
}
🎯
Pattern 4: Remove Nth from End (One Pass)
Gap Pointer Technique

Move a "lead" pointer N steps ahead, then advance both pointers together. When lead reaches end, "trail" is at the target node. One pass, no second traversal.

RemoveNthFromEnd.java
public Node<Integer> removeNthFromEnd(Node<Integer> head, int n) {
    Node<Integer> dummy = new Node<>(0);
    dummy.next = head;
    Node<Integer> lead  = dummy;
    Node<Integer> trail = dummy;

    // Move lead n+1 steps ahead (to create a gap of n)
    for (int i = 0; i <= n; i++) lead = lead.next;

    // Move both until lead hits null
    while (lead != null) {
        lead  = lead.next;
        trail = trail.next;
    }
    trail.next = trail.next.next;  // skip the target node
    return dummy.next;
}
🪞
Pattern 5: Palindrome Check
Find Middle + Reverse + Compare

Combines Pattern 1 (find middle) and Pattern 2 (reverse). Find the middle, reverse the second half, compare both halves.

PalindromeCheck.java
public boolean isPalindrome(Node<Integer> head) {
    if (head == null || head.next == null) return true;

    // Step 1: Find middle (slow/fast pointers)
    Node<Integer> slow = head, fast = head;
    while (fast.next != null && fast.next.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }

    // Step 2: Reverse second half
    Node<Integer> secondHalf = reverse(slow.next);

    // Step 3: Compare first and reversed second half
    Node<Integer> p1 = head, p2 = secondHalf;
    while (p2 != null) {
        if (!p1.data.equals(p2.data)) return false;
        p1 = p1.next; p2 = p2.next;
    }
    return true;
}
⭕
Pattern 6: Find Cycle Start Node
Floyd's Phase 2

After detecting a cycle (slow == fast), reset one pointer to head. Move both one step at a time. Where they meet again is the start of the cycle.

FindCycleStart.java
public Node<Integer> findCycleStart(Node<Integer> head) {
    Node<Integer> slow = head, fast = head;

    // Phase 1: Detect cycle
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) break;  // cycle detected
    }

    // Phase 2: Find the start of the cycle
    slow = head;  // reset slow to head
    while (slow != fast) {
        slow = slow.next;  // both move 1 step
        fast = fast.next;
    }
    return slow;  // this is the cycle start node
}

// 05 — DATA ENGINEERING CONTEXT

Real-World DE Correlations

How linked list concepts directly map to data engineering systems you work with every day.

🔀 Kafka / Message Queue

Singly Linked List = Log Segment Chain

Kafka's partition log is essentially a linked list of segments. Each segment points to the next offset. Consumers traverse from their last committed offset — same as traversing from a node. The "tail" is where new messages are appended in O(1).

🔃 Spark Shuffle / Sort

Merge Pattern = External Merge Sort

When Spark merges sorted partitions during a sort operation, it uses the exact two-sorted-list merge algorithm. Each partition is a "linked list" of records sorted in order. The merge step creates the final sorted output stream.

🔄 DAG / Pipeline

Linked List = Linear Pipeline Stage

An ETL pipeline (Extract → Transform → Load) is a singly linked list of stages. Each stage processes data and passes it to the next. Airflow DAGs are a generalized graph, but single-path pipelines are linked lists at their core.

💾 LRU Cache (Redis)

Doubly Linked List + HashMap

Redis's LRU eviction policy uses a doubly linked list. Recently accessed items move to the head. The least recently used item is always at the tail and gets evicted first. The hashmap provides O(1) lookup; the doubly linked list provides O(1) move-to-head.

LRUCache.java — Classic DE Interview Problem
// LRU Cache = HashMap + Doubly Linked List
// Used in: Redis eviction, database buffer pool, CDN caching
import java.util.*;

class LRUCache {
    private final int capacity;
    private final Map<Integer, DNode> map = new HashMap<>();
    private final DNode head = new DNode(0, 0);  // dummy head
    private final DNode tail = new DNode(0, 0);  // dummy tail

    LRUCache(int capacity) {
        this.capacity = capacity;
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        if (!map.containsKey(key)) return -1;
        DNode node = map.get(key);
        moveToFront(node);   // mark as recently used
        return node.val;
    }

    public void put(int key, int val) {
        if (map.containsKey(key)) {
            DNode n = map.get(key); n.val = val; moveToFront(n); return;
        }
        if (map.size() == capacity) {
            map.remove(tail.prev.key);
            remove(tail.prev);            // evict LRU (tail side)
        }
        DNode node = new DNode(key, val);
        map.put(key, node);
        addToFront(node);                 // most recently used = near head
    }

    private void remove(DNode n) {
        n.prev.next = n.next;
        n.next.prev = n.prev;
    }

    private void addToFront(DNode n) {
        n.next = head.next; n.prev = head;
        head.next.prev = n; head.next = n;
    }

    private void moveToFront(DNode n) { remove(n); addToFront(n); }

    static class DNode {
        int key, val;
        DNode prev, next;
        DNode(int k, int v) { key = k; val = v; }
    }
}

When to Use What

✅

Use Linked List when: You need frequent insertions/deletions at head. You don't know the size upfront. You're building stacks, queues, or deques. You're implementing LRU Cache, undo history, or playlist management.

⚠️

Use Array when: You need random access by index. Data is fixed size or rarely changes. You need cache efficiency (CPU prefetching). You need binary search.
