// 00 — Prerequisite Topics

Build Your Foundation First

You need these concepts solid before stacks "click." Each term below is explained in plain English — with a Data Engineer analogy alongside every definition.

📦 Abstract Data Type (ADT)

An ADT defines what operations exist but not how they're stored. A Stack is an ADT — it just promises push/pop/peek. The internals can be an array or linked list.

🔼 LIFO Principle

Last In, First Out. Whatever was pushed most recently is popped first. Think of a stack of plates — you always take from the top. Contrast with Queue (FIFO = first in, first out).

📐 Call Stack

Your JVM uses a built-in stack to track function calls. When method A calls B calls C — C is on top. When C returns, it's popped and execution returns to B. StackOverflowError = stack is full.

🔁 Recursion

A function that calls itself. Every recursive call adds a new frame to the call stack. Stacks and recursion are deeply related — any recursive algorithm can be rewritten with an explicit stack.

🧩 Interface vs Class

An interface in Java is a contract (e.g., "must have push/pop"). A class is the actual implementation. Deque is Java's stack interface; ArrayDeque is its fastest implementation.

📏 Big-O Notation

O(1) = constant time (instant, regardless of size). All core stack operations — push, pop, peek — run in O(1). That's the whole reason stacks are so powerful.

🏗️ Dynamic Array

An array that resizes itself when full (like Java's ArrayList). Array-backed stacks use this underneath. Resizing is O(n) but amortized O(1) across many pushes.

🔗 Linked List Nodes

The other way to back a stack. Each element is a Node with data + a pointer to the one below. Head = top of stack. Push = add to head (O(1)). Pop = remove head (O(1)).

⚙

Data Engineer parallel: Think of an undo log in a database transaction. Every operation is pushed to the log. On rollback, operations are undone in reverse order — last operation undone first. That's LIFO in production databases.

// 01 — Foundation

What Exactly is a Stack?

A stack is a linear data structure that follows the LIFO rule. You can only interact with the top element — you can't reach into the middle.

🍽 Analogy 1

Stack of Plates

You add plates to the top. You remove from the top. You can't safely grab from the bottom without removing everything above. Same rule governs every stack in computing.

🔙 Analogy 2

Browser Back Button

Every page you visit is pushed onto a history stack. Clicking "back" pops the current page and reveals the previous one. Closing a tab clears the stack. This is a doubly linked list, but the stack metaphor is exact.

▸ Stack Structure — Only the TOP is accessible

Every Term Explained

PUSH

Add an element to the top of the stack. This is the only way to insert. O(1) always. Like placing a new plate on top of the pile.

POP

Remove and return the top element. Throws EmptyStackException if the stack is empty. O(1). You lift the top plate off.

PEEK / TOP

Look at the top element without removing it. Like checking what's on top of the pile without touching it. O(1).

isEmpty

Check if the stack has zero elements. Always check before popping to avoid exceptions. O(1). Like asking "are there any plates left?"

size / height

Total number of elements currently in the stack. Tracked with a counter variable; O(1) to read. Also called the "height" of the stack.

Overflow / Underflow

Overflow = pushing to a full fixed-size stack. Underflow = popping from an empty stack. Java's dynamic stack never overflows (until heap is full).

// 02 — Implementation

Building a Stack — Two Ways

A stack can be backed by an array or a linked list. Same behavior, different internals. Understanding both makes you dangerous in interviews.

▸ Array-backed Stack vs Linked-List-backed Stack
ArrayStack.java — Array-backed implementation
// ── ARRAY-BACKED STACK ──────────────────────────────────────────────
// Uses an int[] internally. Top pointer tracks the index of the top element.

public class ArrayStack<T> {

    private Object[] data;   // internal storage
    private int top = -1;    // -1 means empty; points to current top index
    private int capacity;

    public ArrayStack(int capacity) {
        this.capacity = capacity;
        this.data = new Object[capacity];
    }

    // PUSH — O(1) amortized
    public void push(T item) {
        if (top == capacity - 1) resize();  // double size when full
        data[++top] = item;                   // increment top, then store
    }

    // POP — O(1) — remove and return top element
    @SuppressWarnings("unchecked")
    public T pop() {
        if (isEmpty()) throw new RuntimeException("Stack underflow!");
        T item = (T) data[top];
        data[top--] = null;   // null out to help GC, then decrement top
        return item;
    }

    // PEEK — O(1) — view top without removing
    @SuppressWarnings("unchecked")
    public T peek() {
        if (isEmpty()) throw new RuntimeException("Stack is empty!");
        return (T) data[top];
    }

    public boolean isEmpty() { return top == -1; }
    public int     size()    { return top + 1; }

    private void resize() {
        capacity *= 2;
        data = Arrays.copyOf(data, capacity);  // double capacity
    }
}

// Quick test
ArrayStack<Integer> s = new ArrayStack<>(4);
s.push(10); s.push(20); s.push(30);
System.out.println(s.peek());   // 30 — not removed
System.out.println(s.pop());    // 30 — removed
System.out.println(s.size());   // 2
JavaBuiltIn.java — The Recommended Production Way
// Java's RECOMMENDED stack: use ArrayDeque, NOT Stack class
// Stack class is legacy (extends Vector, which is synchronized = slow)
// ArrayDeque is ~2x faster with no sync overhead

import java.util.ArrayDeque;
import java.util.Deque;

Deque<Integer> stack = new ArrayDeque<>();

stack.push(10);    // push — adds to FRONT (which is top)
stack.push(20);
stack.push(30);

stack.peek();     // → 30  (top element, not removed)
stack.pop();      // → 30  (removes and returns top)
stack.isEmpty(); // → false
stack.size();     // → 2

// Safe pattern: always check before pop
while (!stack.isEmpty()) {
    System.out.println(stack.pop());  // prints: 20, then 10
}

// ── QUICK COMPARISON ────────────────────────────────────────────────
// java.util.Stack   → Legacy. Synchronized. Avoid unless forced.
// ArrayDeque        → Fast. Not thread-safe. Use this by default.
// LinkedList        → Works but more overhead. Use ArrayDeque instead.

The 7 Essential Stack Patterns

These are the reusable blueprints for solving stack problems. Recognize the pattern → the code writes itself.

🔢
Pattern 1: Balanced Brackets / Valid Parentheses CLASSIC

Push every opening bracket. When you see a closing bracket, pop the stack and check if it matches. At the end, the stack must be empty for balance. This exact algorithm powers IDE syntax checkers, JSON validators, and SQL parsers.

▸ Tracing "{[()]}" through the algorithm
ValidParentheses.java
public boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();

    for (char c : s.toCharArray()) {
        // Push opening brackets
        if (c == '(' || c == '{' || c == '[') {
            stack.push(c);
        } else {
            // Closing bracket: stack must not be empty AND must match
            if (stack.isEmpty()) return false;
            char top = stack.pop();
            if (c == ')' && top != '(') return false;
            if (c == '}' && top != '{') return false;
            if (c == ']' && top != '[') return false;
        }
    }
    return stack.isEmpty();  // must be empty at the end
}
// "()[]{}}" → false  |  "{[()]}" → true  |  "([)]" → false
TimeO(n) SpaceO(n) UseJSON/XML validation, SQL parsers
📈
Pattern 2: Monotonic Stack POWERFUL

A stack that is always kept in sorted order (increasing or decreasing). When a new element violates the order, pop until it doesn't. Used for Next Greater Element, stock span, histogram problems. Classic in time-series data analysis.

▸ Next Greater Element — [2, 1, 5, 3, 6, 4] → [5, 5, 6, 6, -1, -1]
NextGreaterElement.java
public int[] nextGreater(int[] nums) {
    int[] result = new int[nums.length];
    Arrays.fill(result, -1);              // default: no greater element
    Deque<Integer> stack = new ArrayDeque<>(); // stores INDICES

    for (int i = 0; i < nums.length; i++) {
        // While stack has indices AND current element is GREATER than top
        while (!stack.isEmpty() && nums[i] > nums[stack.peek()]) {
            int idx = stack.pop();       // this index found its answer
            result[idx] = nums[i];        // current element is the NGE
        }
        stack.push(i);  // push current index — waiting for its NGE
    }
    return result;
}
// [2,1,5,3,6,4] → [5,5,6,6,-1,-1]
// Key: store INDICES in stack, not values — lets you fill the result array
TimeO(n) SpaceO(n) UseStock prices, rainfall, temperature
🧮
Pattern 3: Expression Evaluation TWO STACKS

Use two stacks — one for operands (numbers), one for operators. When an operator of lower precedence is seen, pop and compute from the top. This is how compilers evaluate arithmetic. DE relevance: SQL query engines, formula parsers, Spark expression trees.

EvalRPN.java — Reverse Polish Notation (Postfix)
// Evaluate Reverse Polish Notation: ["2","1","+","3","*"] → 9
// Means: (2 + 1) * 3 = 9
// RPN is what Spark's physical plan uses internally for expressions

public int evalRPN(String[] tokens) {
    Deque<Integer> stack = new ArrayDeque<>();

    for (String token : tokens) {
        switch (token) {
            case "+" -> stack.push(stack.pop() + stack.pop());
            case "-" -> { int b = stack.pop(), a = stack.pop(); stack.push(a - b); }
            case "*" -> stack.push(stack.pop() * stack.pop());
            case "/" -> { int b = stack.pop(), a = stack.pop(); stack.push(a / b); }
            default  -> stack.push(Integer.parseInt(token));  // number
        }
    }
    return stack.pop();  // final result
}
// Trace: ["2","1","+","3","*"]
// push 2 → [2]
// push 1 → [2,1]
// "+"  → pop 1 and 2, push 3  → [3]
// push 3 → [3,3]
// "*"  → pop 3 and 3, push 9  → [9]  → return 9
TimeO(n) SpaceO(n) DE UseSQL engines, Spark physical plans

Pattern 4: Min Stack (O(1) getMin) DESIGN PROBLEM

A stack that supports getMin() in O(1) time. Use a second "shadow" stack that tracks the current minimum at every state. Whenever you push, also push to the min-stack only if the new value ≤ current min.

MinStack.java
class MinStack {
    private final Deque<Integer> stack    = new ArrayDeque<>();
    private final Deque<Integer> minStack = new ArrayDeque<>();

    public void push(int val) {
        stack.push(val);
        // Push to minStack if it's empty OR val ≤ current min
        if (minStack.isEmpty() || val <= minStack.peek())
            minStack.push(val);
    }

    public void pop() {
        int val = stack.pop();
        // Remove from minStack only if it was the current min
        if (val == minStack.peek()) minStack.pop();
    }

    public int top()    { return stack.peek(); }
    public int getMin() { return minStack.peek(); }  // O(1) !
}
// push(5) → stack:[5]    minStack:[5]
// push(3) → stack:[3,5]  minStack:[3,5]
// push(7) → stack:[7,3,5] minStack:[3,5]   ← 7 > 3, not pushed to minStack
// getMin()→ 3  (peek minStack)
// pop()   → stack:[3,5]  minStack:[3,5]    ← 7 removed, not in minStack
// getMin()→ 3  still correct!
All opsO(1) SpaceO(n) DE UseSliding window min, stream stats

Pattern 5: Iterative DFS / Tree Traversal RECURSION → STACK

Any recursive DFS can be converted to an iterative version using an explicit stack. Push right child before left child so left is processed first (LIFO!). Critical for environments with stack size limits — like massive Spark job DAGs.

IterativeDFS.java — Preorder Tree Traversal
public List<Integer> preorder(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;

    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);

    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        result.add(node.val);                   // visit node

        // Push RIGHT first so LEFT is processed first (LIFO!)
        if (node.right != null) stack.push(node.right);
        if (node.left  != null) stack.push(node.left);
    }
    return result;  // Root → Left → Right (Preorder)
}

// Why push RIGHT before LEFT?
// Stack is LIFO — last pushed = first popped.
// We want to pop LEFT first → so push RIGHT first.
// This is the core "reversal" insight of stack-based DFS.
TimeO(n) SpaceO(h) Noteh = height of tree

Pattern 6: Decode / Nested String Expansion TWO STACKS

When you hit a [, save current state to stack and start fresh. When you hit ], pop the saved state and combine. Used in template engines, configuration expansion, and nested JSON processing — all common in DE pipelines.

DecodeString.java — "3[a2[c]]" → "accaccacc"
public String decodeString(String s) {
    Deque<Integer> countStack  = new ArrayDeque<>();  // stores repeat counts
    Deque<String>  stringStack = new ArrayDeque<>();  // stores built strings
    String current = "";
    int k = 0;

    for (char c : s.toCharArray()) {
        if (Character.isDigit(c)) {
            k = k * 10 + (c - '0');         // handle multi-digit numbers
        } else if (c == '[') {
            countStack.push(k);              // save count
            stringStack.push(current);       // save current string
            current = ""; k = 0;            // reset for inner content
        } else if (c == ']') {
            int repeat = countStack.pop();
            String prev = stringStack.pop();
            current = prev + current.repeat(repeat); // expand and append
        } else {
            current += c;                      // regular char
        }
    }
    return current;
}
// "3[a2[c]]"  →  "accaccacc"
// "2[bc]3[a]" →  "bcbcaaa"
TimeO(n·k) SpaceO(n) DE UseTemplate engines, config expansion

Pattern 7: Undo / Redo with Two Stacks SYSTEM DESIGN

Classic application of two stacks. Every action pushed to undoStack. On undo: pop from undoStack → push to redoStack. On redo: pop from redoStack → push back to undoStack. Used in editors, DB transaction logs, version control staging areas.

UndoRedoSystem.java
class TextEditor {
    Deque<String> undoStack = new ArrayDeque<>();
    Deque<String> redoStack = new ArrayDeque<>();
    String current = "";

    public void type(String text) {
        undoStack.push(current);  // save current state before changing
        redoStack.clear();         // new action clears redo history
        current = current + text;
    }

    public void undo() {
        if (undoStack.isEmpty()) return;
        redoStack.push(current);   // save for redo
        current = undoStack.pop(); // restore previous state
    }

    public void redo() {
        if (redoStack.isEmpty()) return;
        undoStack.push(current);   // save for undo
        current = redoStack.pop(); // restore redo state
    }
}
// type("Hello")  → current="Hello"      undo:[]       redo:[]
// type(" World") → current="Hello World" undo:["Hello"] redo:[]
// undo()         → current="Hello"       undo:[]       redo:["Hello World"]
// redo()         → current="Hello World" undo:["Hello"] redo:[]
All opsO(1) DE UseDB transaction log, ETL rollback

// 05 — Real-World DE Connections

Stacks in Data Engineering

Every major DE tool uses stacks internally. Understanding this makes you a better systems thinker.

🔥 Apache Spark

Catalyst Optimizer — Expression Tree

Spark's SQL engine uses a stack to evaluate expression trees in its Catalyst Optimizer. When it parses SELECT a + b * c, it builds an AST and uses a stack to evaluate operator precedence — exactly Pattern 3.

🌊 Apache Flink

Checkpoint & Rollback

Flink's checkpointing mechanism pushes state snapshots to a log. On failure, it pops back to the last consistent checkpoint. This is the undo-stack pattern applied to distributed streaming pipelines.

🗄 PostgreSQL

Transaction Savepoints

Database transaction savepoints are a stack. SAVEPOINT sp1 = push. ROLLBACK TO sp1 = pop back. RELEASE sp1 = commit and discard. Nested transactions literally use a stack structure.

🔀 Apache Airflow

DAG Task Scheduling (DFS)

Airflow uses Depth-First Search (stack-based) to resolve task dependencies when scheduling. It traverses the DAG and pushes tasks onto a stack, executing in topological order — Pattern 5 at scale.

📋 dbt (data build tool)

Ref() Resolution & Lineage

When dbt compiles your model graph, it resolves ref() calls using a DFS traversal with a stack to detect cycles and build execution order. If a circular ref exists, the stack detects it (like cycle detection in linked lists).

🔍 JSON / Parquet Parsing

Nested Structure Parsing

Parsing nested JSON or Parquet schema uses the bracket-matching pattern. Every { or [ pushes context onto a stack. Every } or ] pops and validates — Pattern 1 + Pattern 6 combined.

SparkLikeExpressionEval.java — How Spark evaluates column expressions
// Simplified version of how Spark's Catalyst evaluates:
// col("revenue") - col("cost") * 0.1
// represented as RPN: ["revenue", "cost", "0.1", "*", "-"]

public double evalColumnExpr(String[] tokens, Map<String, Double> row) {
    Deque<Double> stack = new ArrayDeque<>();

    for (String token : tokens) {
        if (row.containsKey(token)) {
            stack.push(row.get(token));        // column value from row
        } else if (token.matches("-?\\d+(\\.\\d+)?")) {
            stack.push(Double.parseDouble(token)); // literal number
        } else {
            double b = stack.pop(), a = stack.pop();
            switch (token) {
                case "+" -> stack.push(a + b);
                case "-" -> stack.push(a - b);
                case "*" -> stack.push(a * b);
                case "/" -> stack.push(a / b);
            }
        }
    }
    return stack.pop();
}

// Usage:
// Map row = {"revenue": 1000.0, "cost": 200.0}
// tokens  = ["revenue", "cost", "0.1", "*", "-"]
// Result  = 1000 - (200 * 0.1) = 980.0
// This is how Spark evaluates column transformations row by row!

✅

Use a Stack when: Order of processing must be reversed. You need to track "pending" work while diving deeper. You're implementing any DFS, undo, bracket matching, or expression parsing. The last item added should be the first processed.

⚠️

Avoid a Stack when: You need FIFO order (use a Queue). You need random access by index (use an ArrayList). You need sorted access (use a TreeSet/PriorityQueue). You need bidirectional traversal (use Deque as queue).
