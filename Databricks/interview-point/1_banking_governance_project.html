What This Project Covers
🔐 Security Layer
  • Azure Private Endpoint — no public internet for SQL/ADLS
  • Managed Identity — passwordless auth everywhere
  • Azure Key Vault — all secrets via dbutils.secrets.get()
  • TLS in-transit + AES-256 at-rest encryption
🔍 Data Lineage
  • Unity Catalog auto-captures lineage on every table write
  • Microsoft Purview tracks cross-platform lineage (SQL→ADLS→DBX)
  • Queryable via system.access.table_lineage

  • RBI NPA classification logic in Gold layer
  • Column-level masking policies per user group
  • Row filters — branch managers see only their branch
  • KYC renewal tracking & compliance dashboards
📊 Business Domains
  • Fraud Domain — daily transaction risk scoring
  • Risk Domain — NPA classification, credit risk profiles
  • Compliance Domain — KYC compliance dashboard
  • Lending Domain — loan portfolio health metrics

🥉 Layer 1

Bronze Layer — Raw Ingestion

Land raw data as-is into Delta tables. Add audit metadata. No business transformation.

💡
Design Principle: Bronze is immutable and append-only. It is the single source of truth. If anything goes wrong downstream, we can always replay from Bronze.
Key Code — Bronze Notebook
01_bronze_ingestion.pyPySpark
# Step 1: Retrieve ADLS key from Key Vault (zero hardcoding)
storage_key = dbutils.secrets.get(scope="banking-kv-scope", key="adls-storage-key")
spark.conf.set(f"fs.azure.account.key.bankdatalake.dfs.core.windows.net", storage_key)

# Step 2: Read raw parquet dropped by ADF
raw_df = spark.read.format("parquet").load(
    "abfss://[email protected]/transactions/2024/01/15/"
)

# Step 3: Add audit metadata (governance requirement)
bronze_df = raw_df \
    .withColumn("_ingestion_timestamp", current_timestamp()) \
    .withColumn("_source_system",       lit("AzureSQL_BankingCoreDB")) \
    .withColumn("_pipeline_name",       lit(f"ADF_banking_daily|{PIPELINE_ID}")) \
    .withColumn("_source_file",         input_file_name())

# Step 4: Write to Unity Catalog — lineage auto-captured!
bronze_df.write.format("delta").mode("append") \
    .partitionBy("transaction_date") \
    .saveAsTable("banking_catalog.bronze_layer.transactions_bronze")

Data Quality Pattern
databricks_utils.py — DQ CheckPySpark
def run_dq_checks(df, table_name, not_null_cols, primary_key=None):
    # Split into clean vs quarantine based on null checks
    null_condition = reduce(lambda a,b: a|b,
        [col(c).isNull() for c in not_null_cols if c in df.columns])

    quarantine_df = df.filter(null_condition) \
                     .withColumn("_dq_fail_reason", lit("null_in_required_column"))
    clean_df      = df.filter(~null_condition)

    # Deduplication using window function on primary key
    if primary_key:
        window = Window.partitionBy(primary_key).orderBy(desc("_ingestion_timestamp"))
        clean_df = clean_df.withColumn("_rn", row_number().over(window)) \
                           .filter(col("_rn") == 1).drop("_rn")

    return clean_df, quarantine_df   # Quarantine goes to audit_layer
🥇 Layer 3

Gold Layer — Business Aggregations

Domain-specific tables ready for Power BI, ML models, and Risk APIs.

🚨 fraud_daily_txn_summary
  • Consumers: Fraud team, real-time alert API
  • Key Fields: fraud_risk_score, fraud_alert_flag, high_value_txn_count
  • Logic: Rule-based composite score (replace with ML model in production)
  • Access: fraud_analysts group only
📉 customer_risk_profile
  • Consumers: Risk dashboards, Basel III reporting
  • Key Fields: npa_classification, max_days_past_due, total_exposure
  • Logic: RBI DPD rules — DPD ≥ 90 = NPA (Sub-Standard)
  • Access: risk_analysts group
✅ kyc_compliance_dashboard
  • Consumers: Compliance portal, RBI audit submissions
  • Key Fields: kyc_compliance_rate, compliance_status, renewal_due_count
  • Logic: Rate ≥ 95% = COMPLIANT, ≥ 80% = AT_RISK, else NON_COMPLIANT
  • Access: compliance_officers group
🏦 loan_portfolio_health
  • Consumers: Lending dashboard, provisioning models
  • Key Fields: npa_rate, overdue_rate, portfolio_health
  • Logic: NPA rate ≤ 2% = HEALTHY, ≤ 5% = WATCH, else STRESSED
  • Access: risk_analysts + lending team
🎯
Production tip: Gold tables are rebuilt daily using replaceWhere for the current date partition — not full table overwrites. This makes reruns fast and safe even for 100M+ row tables.

Column Masking Policy (SQL)
unity_catalog_governance_setup.sqlSQL
-- Dynamic masking: same table, different views per group
CREATE OR REPLACE FUNCTION banking_catalog.silver_layer.mask_account_policy(acct STRING)
  RETURN CASE
    WHEN is_member('fraud_analysts')      THEN acct          -- See hashed value
    WHEN is_member('risk_analysts')       THEN acct
    WHEN is_member('compliance_officers') THEN acct
    ELSE '****-****-****'                              -- Fully masked
  END;

ALTER TABLE banking_catalog.silver_layer.transactions_silver
  ALTER COLUMN account_number
  SET MASK banking_catalog.silver_layer.mask_account_policy;

-- Row filter: branch managers see only their branch
ALTER TABLE banking_catalog.gold_layer.kyc_compliance_dashboard
  SET ROW FILTER banking_catalog.gold_layer.branch_row_filter ON (branch_code);
Audit Query — Compliance Report
Audit: Who accessed Silver tables (last 30 days)SQL
SELECT
  event_time,
  user_identity.email            AS user_email,
  request_params.table_full_name AS table_accessed,
  action_name,
  source_ip_address
FROM system.access.audit
WHERE
  action_name IN ('selectQuery', 'runCommand')
  AND request_params.table_full_name LIKE 'banking_catalog.silver%'
  AND event_time >= current_date - INTERVAL 30 DAYS
ORDER BY event_time DESC;
🔭 Microsoft Purview

Purview — Enterprise Catalog + Lineage

Cross-platform: Azure SQL → ADLS → Databricks → Gold — full lineage in one graph

🔗
Key Difference: Unity Catalog governs within Databricks. Purview governs across all Azure services. Together they give complete end-to-end coverage.
🏷️ Auto PII Classification

Purview scans all registered sources and auto-detects: INDIA_PAN, INDIA_AADHAAR_NUMBER, BANK_ACCOUNT_NUMBER, EMAIL_ADDRESS, PHONE_NUMBER_IN using built-in classifiers. No manual tagging needed.

🗺️ Cross-Platform Lineage

Single lineage graph from Azure SQL all the way to Gold tables: dbo.transactions → ADLS raw → Bronze → Silver → Gold → Power BI. Queryable via REST API or Purview UI.

📖 Business Glossary

Define business terms like dpd_days (Days Past Due, RBI DPD ≥ 90 = NPA), npa_classification, fraud_risk_score — linked to actual columns across all systems.

🔐 Sensitivity Labels

Tables classified as: Restricted (Bronze raw PII), Confidential (Silver masked), Internal (Gold aggregations). Labels flow into Azure Policy for residency enforcement.
